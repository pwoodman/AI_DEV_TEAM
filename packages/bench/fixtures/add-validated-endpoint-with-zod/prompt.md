Add `POST /items` to `src/router.ts`. It must parse the request body with a zod schema (fields: `name: string`, `qty: number` non-negative), assign a UUID-like id (`crypto.randomUUID()`), append the created item to the in-memory `store` (exported from `src/store.ts`), and return status 201 with the created item. Invalid bodies (missing `name` or negative `qty`) must return status 400.
