## Notable observations

**superpowers tokensCached is very large:**
- `add-cli-flag`: 512,342 cache-read tokens
- `add-typed-error`: 175,616 cache-read tokens
- `fix-failing-vitest`: 353,994 cache-read tokens

Claude Code ships a massive cached system prompt (~512K tokens on first task, warm on subsequent). These cache-read tokens do not count toward `tokensIn`/`tokensOut` in our comparison (they're billed at ~10% of normal input cost). The `tokensIn` for superpowers was only 13–30 per task because almost everything hit cache; the `tokensOut` (1408, 554, 926) is where superpowers does its work. AMASE's pattern is inverted: higher `tokensIn` (structured context), much lower `tokensOut` (focused patches).

**diffSimilarity:**
- `fix-failing-vitest`: 0.65 (both stacks produced similar patches — the fix is obvious)
- `add-cli-flag` / `add-typed-error`: ~0.26 (different patch shapes, both passing tests)
